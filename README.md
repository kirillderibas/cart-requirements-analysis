# Cart Requirements Analysis — "Петрушка Зеленая"

Задание №1: анализ фрагмента ТЗ по функционалу корзины интернет-магазина.

## Что внутри
- `docs/01_issues.md` — найденные логические противоречия и недочеты с объяснениями
- `docs/02_fixed_spec.md` — исправленная, логически непротиворечивая версия требований
- `docs/03_questions.md` — уточняющие вопросы к продукт-менеджеру, заказчику
- `docs/04_acceptance_criteria.md` — критерии приемки и базовые сценарии проверки

## Ключевое проектное решение
В исходном ТЗ есть конфликт требований по цене (фиксировать при добавлении vs автоматически обновлять при изменении в каталоге).
В исправленной версии выбрана модель: **в корзине отображается актуальная цена из каталога** с пересчетом при открытии корзины и перед оформлением заказа.

Альтернативная модель (фиксация цены в корзине) описана в `docs/03_questions.md` как вариант для согласования с бизнесом.
